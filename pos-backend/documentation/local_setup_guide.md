# 💻 Lumora POS - Local Setup Guide

This guide provides step-by-step instructions for developers to set up and run the Lumora Enterprise POS system on their local development environment.

## 🛠️ 1. Prerequisites
Ensure you have the following installed on your machine:
- **Java Development Kit (JDK) 17** (or higher)
- **Node.js** (v18.x or v20.x recommended)
- **PostgreSQL** (v14 or higher)
- **Git**

---

## 🗄️ 2. Database Configuration
The backend relies on PostgreSQL and automatically manages schema migrations using Flyway.

1. Open your PostgreSQL terminal (psql) or a tool like pgAdmin / DBeaver.
2. Create the development database:
   ```sql
   CREATE DATABASE lumora_pos;
   ```
3. Set up the local user (if you aren't using the default `postgres` user):
   ```sql
   CREATE USER lumora_user WITH PASSWORD 'lumora_pass';
   GRANT ALL PRIVILEGES ON DATABASE lumora_pos TO lumora_user;
   ```

---

## ⚙️ 3. Backend Setup (Spring Boot)
The backend is a monolithic Spring Boot application managing the API, security, and database interactions.

1. Open a terminal and navigate to the backend directory:
   ```bash
   cd "backend"
   ```

2. **Environment Variables Config:**
   The application uses `application-dev.yml` by default for local development. Ensure the credentials in this file match your local PostgreSQL setup. You can also override them via environment variables:
   - `DB_USERNAME` (default falls back to `postgres`)
   - `DB_PASSWORD` (default falls back to `postgres`)
   - `JWT_SECRET` (A default 256-bit safe secret is provided for dev, but can be overridden)

3. **Install Dependencies and Run Migrations:**
   Spring Boot will automatically run Flyway migrations (`V1` to `V27`) on the first startup, creating tables and seeding default data.
   ```bash
   ./mvnw clean install -DskipTests
   ```

4. **Start the Backend Server:**
   ```bash
   ./mvnw spring-boot:run
   ```
   *The backend will be available at `http://localhost:8081`.*
   *API Documentation (Swagger) is available at `http://localhost:8081/swagger-ui.html`.*

---

## 🎨 4. Frontend Setup (Next.js)
The frontend is a React application built on the Next.js App Router.

1. Open a **new** terminal window and navigate to the frontend directory:
   ```bash
   cd "frontend"
   ```

2. **Install Dependencies:**
   ```bash
   npm install
   # or
   yarn install
   ```

3. **Environment Setup:**
   Ensure your `.env.local` or `.env` file contains the API URL mapping. Create `.env.local` if it doesn't exist:
   ```env
   NEXT_PUBLIC_API_URL=http://localhost:8081/api/v1
   ```

4. **Start the Development Server:**
   ```bash
   npm run dev
   # or
   yarn dev
   ```
   *The frontend will be available at `http://localhost:3000`.*

## 🚨 Troubleshooting
- **Port Conflicts:** If `8081` or `3000` is in use, find the ghost process and kill it (e.g. `npx kill-port 8081` or using OS task manager).
- **Flyway Errors:** If a migration fails mid-way during development schema changes, run `./mvnw flyway:repair` from the backend directory to clear the lock.
- **Rate Limiting:** Auth endpoints enforce strict rate limits (10 attempts / 15 mins per IP). If locked out, restart the backend server to clear the in-memory cache limit.
